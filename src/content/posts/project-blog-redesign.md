这次重建博客时，目标不是单纯换一个框架，而是让内容结构更接近真实写作。

旧的文档站适合放学习笔记，但首页表达能力有限，栏目之间也不够清楚。新的结构把“栏目”作为一等入口。

## 为什么用 Astro

Astro 适合内容型站点：静态输出、Markdown 友好、组件化足够轻，部署到 GitHub Pages 也简单。

## 为什么先做栏目

如果只有文章列表，内容多了以后会变成时间流。栏目可以让读者先按兴趣进入，也让作者知道每篇文章属于哪种写作。

## 保留自动部署

GitHub Actions 继续负责安装依赖、构建和发布。这样本地只需要关注写作和预览，发布流程保持稳定。

这个项目后续最值得继续改的是搜索、评论和更完整的主题设计。
