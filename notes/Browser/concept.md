## 🚀 主题一：浏览器的工作原理

首先，我们来了解浏览器是如何把你的代码变成用户眼中的漂亮页面的。这个过程的核心就是**关键渲染路径（Critical Rendering Path, CRP）**。

简单来说，当你输入一个网址并按下回车，浏览器会经历以下几个关键步骤：

1.  **加载 (Loading)**：从网络上获取 HTML 字节数据。
2.  **解析 (Parsing)**：
    * **构建 DOM 树 (Document Object Model)**：解析 HTML 标记，构建一个表示页面结构的对象模型。
    * **构建 CSSOM 树 (CSS Object Model)**：解析所有 CSS (包括内联、内部和外部样式)，构建一个表示样式规则的对象模型。
3.  **构建渲染树 (Render Tree)**：将 DOM 树和 CSSOM 树结合起来。渲染树只包含**可见的元素**及其**计算后的样式**。（**注意：** 像 `<head>` 或 `display: none` 的元素是不会进入渲染树的哦！）
4.  **布局 (Layout/Reflow)**：根据渲染树，计算每个可见元素在屏幕上的确切位置和大小。
5.  **绘制 (Painting)**：用像素填充元素，包括文本、颜色、边框、阴影等。
6.  **合成 (Compositing)**：将所有绘制好的图层（Layers）合并，最终显示在屏幕上。
