## 🎯 开发优先级

### 高优先级任务
1. **类型安全**: 确保所有 TypeScript 类型定义完整
2. **测试覆盖**: 单元测试覆盖率 >= 85%
3. **代码规范**: 严格遵循 代码风格规范.md
4. **文档同步**: 代码变更同步更新相关文档

### 质量检查清单
在完成开发任务后，必须检查：
- [ ] 代码通过类型检查 (`pnpm run typecheck`)
- [ ] 所有测试通过 (`pnpm run test`)
- [ ] 代码符合规范 (`pnpm run lint`)
- [ ] 组件有完整的测试覆盖
- [ ] 文档已更新
- [ ] 提交信息符合规范

## ⚠️ 重要提醒

### 开发注意事项
1. **始终使用 TypeScript**，避免使用 `any` 类型
2. **遵循严格的代码规范**，确保团队代码一致性
3. **编写完整的测试**，确保代码质量和稳定性
4. **保持文档更新**，确保文档与代码同步

### 提交前检查清单
完成代码修改后，务必运行：
- `pnpm run typecheck` - 类型检查，确保无类型错误
- `pnpm run test` - 运行所有测试，确保测试通过
- `pnpm run lint` - 代码检查，确保符合规范
- `pnpm run format` - 代码格式化，保持一致性

3. 查看项目的 GitHub Issues
